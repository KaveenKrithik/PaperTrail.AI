# Product Roadmap (Timeline)

| Phase / Release | Tentative Timeline | Objectives | Key Deliverables |
| --- | --- | --- | --- |
| R0: Zeroth Review | Week 0 | Define scope and requirements | Vision statement, backlog, SDG mapping, roadmap |
| R1: Foundation MVP | Weeks 1-4 | End-to-end audit for a single document | Upload + storage, OCR/text extraction, structured extraction, orchestration pipeline, baseline report |
| R2: Trust & Accuracy | Weeks 5-8 | Improve verification quality | Consistency checks, compliance rules, risk scoring v1, evidence mapping |
| R3: User Confidence | Weeks 9-12 | Make audits explainable and challengeable | Dispute system (counter-auditor), report export, audit history dashboard |
| R4: Differentiation | Weeks 13-14 | Unique product edge | Visual evidence overlays, multi-document cross-checks, email ingestion |

Notes:
- Each phase can be implemented in 2-week sprints.
- Scope is adjustable based on review feedback and resource constraints.
